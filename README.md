# FileManagerInTelegramBot-BETA
Бот для управления файловой структурой в Telegram с возможностью организации файлов и обмена данными.

## Возможности
- Создание и управление папками. Организация файлов в структуру по типу древовидной;
- Расширенная поддержка различных типов файлов: от обычных текстовых сообщений до аудио и видео;
- Возможность публичного доступа к папкам;
- Управление файлами;
- Интуитивный и понятный интерфейс.

## Быстрый старт
### Установка
1. Клонирование репозитария:
```
git clone https://github.com/n0ks-0155/FileManagerInTelegramBot-BETA.git
```
2. Настройка зависимостей:
```
pip install -r requirements.txt
```
3. Настройте файл `config.py`, вставив свой токен.
4. Запустите бота:
```
python bot.py
```
## Использование
### Основные команды
| Команда | Описание |
| --- | --- |
| /start | Запуск бота |
| /mkdir | Создание новой папки |
| /cd | Перейти в папку |
| /up | Вернуться на уровень выше |
| /getmydata | Показать содержимое папки |
| /share | Сделать папку публичной |
| /access | Получить доступ к папке |
### Работа с файлами
1. Отправьте любой файл боту - автоматически сохранится в папке.
2. Используйте кнопки для навигации.
3. Нажмите кнопку удаления рядом с файлом, чтобы удалить его раз и навсегда.
### Обмен доступом
1. Введите команду для того, чтобы сделать папку публичной: `/share`
2. Получите уникальный идентификатор для доступа к папке
3. Передайте ключ другому пользователю
4. Пользователь получает доступ, с помощью команды `/access`
## Структура проекта
```
telegram-file-manager/
├── bot.py                 #Основной файл для запуска бота
├── config.py              #Токен бота
├── user_data.json         #База данных (JSON*)
├── requirements.txt       #Зависимости
├── README.md            
└── handlers/             
    ├── command_handlers.py   #Обработчики команд
    ├── callback_handlers.py  #Обработчики callback-запросов
    └── message_handlers.py   #Обработчики сообщений
└── utils/                
    ├── data_manager.py   #Управление данными
    ├── navigation.py     #Навигация по структуре папок
    └── keyboards.py      #Генерация inline-клавиатур
```
`JSON*` - можно оптимизировать хранение данных (использовать базу данных вместо JSON-файла).

## Ограничения
1. Максимальный размер файла: 50 МБ
2. Возможные лимиты на частоту запросов
3. Все данные хранятся в локальном JSON-файле
4. Пуличное хранение токена для бота
